# Security Policy

We actively support the following versions of the project with security updates. Older versions may no longer receive security patches.

| Version | Supported          |
| ------- | ------------------ |
| v1.x    | :white_check_mark:  |

## Reporting a Vulnerability

If you discover a potential security vulnerability, we encourage you to report it to us responsibly. Please follow the steps below to ensure the issue is addressed promptly.

### How to Report

1. **Email**: Send a detailed description of the vulnerability to support@rndigitaloperations.com.
2. **Details to Include**:
   - A detailed description of the vulnerability
   - The version(s) of the project affected
   - Steps to reproduce the vulnerability
   - Any suggested fixes or mitigations (if applicable)
3. **Confidentiality**: All reported vulnerabilities will be kept confidential until a fix or mitigation is available.

### What to Expect

- **Acknowledgement**: We will acknowledge your report within 2 business day.
- **Investigation**: In most cases, we begin investigating the issue the same day we respond to your report. If there are any delays, investigation will begin the following business day at the latest.
- **Resolution**: Once a vulnerability is verified, we will prioritize a patch and communicate the resolution timeline. We aim to release a patch within 5 days of verification.
- **Notification**: After a patch is released, we will notify you of the availability and post an advisory if necessary.

### Handling Accepted Vulnerabilities

If your reported vulnerability is confirmed:
- We will work with you to create a fix and update any affected versions of the project.
- We will publicly disclose the vulnerability and the corresponding patch details as soon as it is safe to do so.

### Handling Declined Vulnerabilities

If your reported vulnerability is not accepted:
- We will provide an explanation of why the report did not meet our criteria.
- We may recommend other ways to mitigate the issue or suggest alternative solutions.

## Security Updates and Patch Releases

- **Security Fixes**: Security patches will be made available as part of new version releases.
- **Version Support**: Older versions of the project may not receive security updates. We strongly recommend upgrading to a supported version.
- **Emergency Patches**: If a critical security vulnerability is discovered, emergency patches may be released outside of the regular release cycle.

## Security Best Practices

We recommend following these general best practices to secure your use of this project:

- Keep your project dependencies up-to-date to reduce exposure to known vulnerabilities.
- Regularly audit your environment and the services that interact with this project for potential security risks.
- Review and follow the latest security advisories related to your version of the project.

For any security questions or concerns, please contact us at support@rndigitaloperations.com.
